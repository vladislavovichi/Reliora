# Продукт

## Что Такое Reliora

Reliora — внутренний helpdesk для команд, которые хотят вести поддержку внутри Telegram, но не готовы жертвовать качеством процессов ради скорости запуска.

Продукт сочетает два на первый взгляд конфликтующих свойства:

- оператор работает в лёгком Telegram-интерфейсе;
- продукт остаётся достаточно зрелым для архива, аналитики, exports, attachments, feedback и ролей.

## Основной Пользовательский Ритм

### Клиент

1. Открывает чат.
2. Выбирает тему.
3. Описывает ситуацию текстом или прикладывает материалы.
4. Продолжает диалог в том же Telegram-чате.
5. После закрытия может оставить оценку и комментарий.

### Оператор

1. Берёт заявку из очереди или продолжает активный диалог.
2. Отвечает, использует макросы, добавляет заметки и метки.
3. При необходимости передаёт, эскалирует или закрывает дело.
4. Возвращается к истории через архив.
5. Выгружает `HTML` или `CSV`, если нужен review, handoff или audit.

### Суперадминистратор

1. Управляет составом команды.
2. Выдаёт invite-коды новым операторам.
3. Настраивает темы и макросы.
4. Смотрит спокойную аналитику и exports.

## Что Делает Продукт Ценным

- Архив — не свалка закрытых кейсов, а рабочее пространство для повторного просмотра и экспорта.
- HTML-отчёты подходят не только разработчику, но и операционному или управленческому review.
- Operator onboarding больше не требует ручного ввода Telegram ID: супер-админ может выдать одноразовый invite-код.
- Attachment handling и внутренние заметки встроены в общую модель дела.

## Язык Интерфейса

Telegram-facing тексты намеренно остаются:

- русскими;
- короткими;
- спокойными;
- небюрократическими;
- пригодными для ежедневной работы, а не только для демо.
