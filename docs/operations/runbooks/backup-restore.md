# Резервная копия и восстановление PostgreSQL

## Что сохраняется

`make backup-db` делает логическую резервную копию PostgreSQL через `pg_dump -Fc`.

В него попадает всё, что хранится в базе:

- заявки и история сообщений;
- операторы, роли и инвайт-коды;
- темы, теги и макросы;
- обратная связь;
- аудит;
- кэш AI-сводок и остальные таблицы приложения.

## Что не сохраняется

В резервную копию не входят:

- данные Redis;
- локальные файлы в `assets/`, если они лежат вне PostgreSQL;
- `.env`;
- образы контейнеров и Compose-конфигурация.

Иными словами, это резервная копия базы, а не слепок всего окружения.

## Как сделать резервную копию

```bash
make backup-db
```

По умолчанию файл появится в `backups/postgres/` с меткой времени в UTC.

Если нужен свой путь:

```bash
BACKUP_PATH=backups/postgres/release-2026-04-13.dump make backup-db
```

## Как восстановить

```bash
BACKUP_PATH=backups/postgres/helpdesk_20260413T120000Z.dump make restore-db
```

Восстановление использует `pg_restore --clean --if-exists`. Команда перезаписывает объекты в целевой базе и требует аккуратности.

## На что обратить внимание

- лучше выполнять восстановление без активной пользовательской нагрузки;
- важно не перепутать окружение;
- после восстановления нужно сразу проверить стек;
- если restore делается после неудачного релиза, нужно отдельно оценить совместимость кода и схемы.

Минимальная проверка после восстановления:

```bash
make health
make smoke
```
