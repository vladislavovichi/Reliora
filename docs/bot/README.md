# Telegram Bot

## За Что Отвечает Bot-Слой

`src/bot` отвечает за то, как Reliora ощущается в ежедневной работе. Здесь живут:

- русские тексты;
- inline и reply keyboards;
- formatter'ы карточек и списков;
- маршрутизация handler'ов;
- middleware авторизации и update context;
- delivery клиенту и оператору;
- onboarding через invite-коды.

Bot не должен становиться местом, где прячутся бизнес-правила. Его задача — собрать аккуратный экран и передать действие дальше в service/backend слой.

## UX-Направление

Для operator-side UX проект держится нескольких постоянных принципов:

- button-first навигация;
- короткие русские формулировки;
- спокойные экраны без технического шума;
- discoverability без перегруза кнопками;
- уважение к active dialogue и текущему контексту оператора.

Это особенно заметно в архиве, exports и admin surfaces: действия остаются очевидными, но интерфейс не превращается в панель с десятком равнозначных кнопок.

## Основные Разделы

- `Очередь`
- `Мои заявки`
- `Архив`
- `Статистика`
- `Операторы`, `Макросы`, `Темы` для super-admin surfaces

Каждый раздел имеет собственный formatter и собственные keyboard builders. Это намеренно: текст, раскладка кнопок и экранная ритмика меняются независимо.

## Вложения И Диалог

Bot-слой принимает:

- фото;
- документы;
- голосовые;
- видео.

На входе уже действует политика hardening:

- ограничения размера;
- отказ от исполняемых документов;
- нормализация filename / MIME metadata;
- спокойные русские ошибки для unsupported и unsafe cases.

## Архив И Экспорты

Архив оформлен как отдельный browsing flow:

- стабильная строка действий с `Все темы` и отдельным входом в picker тем;
- отдельная поверхность выбора темы;
- архивная карточка дела;
- очевидный экспорт `HTML` / `CSV` из исторической карточки.

Экспорты остаются отдельными transport-действиями, а не побочным эффектом просмотра.
